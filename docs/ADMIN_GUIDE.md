# 管理员使用手册

本手册的目标是让管理员能够尽快完成日常操作，并在出问题时知道先看哪里、怎么回滚。

## 5 分钟上手

1. 打开 `/admin/login`。
2. 输入管理员账号和密码。
3. 登录后进入 `/admin` 控制面板。
4. 从左侧导航进入新闻、活动、校友名单或修改申请页面。
5. 任何删除、导入、批量覆盖操作前，先确认是否已有备份。

## 如何进入后台

1. 访问 `/admin/login`。
2. 输入管理员账号和密码。
3. 登录成功后自动跳转控制面板（`/admin`）。

### 相关环境变量

| 变量名 | 用途 |
|--------|------|
| `ADMIN_USERNAME` | 管理员登录账号 |
| `ADMIN_PASSWORD_HASH` | 管理员密码的 SHA256 哈希值 |
| `SESSION_SECRET` | HMAC-SHA256 token 签名密钥 |

> 以上变量在服务器 `.env` 中配置，不写入代码。`.env` 不提交 git。

### 凭据管理

- 管理员与普通访问口令是两套独立的凭据体系。
- 退出后清除 cookie，需重新登录。
- 普通口令用户无法访问 `/admin/*` 和 `/api/admin/*`。

### 快速修改凭据

如果需要轮换管理员账号密码或访问口令，优先使用仓库内脚本，而不是手工改文件：

```bash
cp credentials.example.json credentials.local.json
node scripts/set-credentials.js
```

- `credentials.local.json` 只保存在本地，不提交到仓库
- 脚本会原子写入并保留备份，出问题可回滚
- 修改后需要重新登录，旧 cookie 会失效

## 新闻管理

路径：`/admin/news`

### 适合什么时候使用

- 发布学校新闻、公告、活动预告
- 下架已过期内容
- 删除误发内容前先确认是否需要保留历史

### 发布新闻

1. 点击"新建新闻"。
2. 填写标题、摘要（可选）、正文内容。
3. 可选上传封面图片（支持 png/jpg/webp，建议尺寸 1200×630）。
4. 点击"保存草稿"或"发布"。

### 编辑新闻

1. 在列表页找到目标新闻。
2. 点击"编辑"进入编辑页。
3. 修改后保存。

### 删除新闻

1. 在列表页点击删除按钮。
2. 二次确认弹窗，确认后删除。
3. 删除不可撤销。

### 发布/下架

列表页切换状态开关，控制前台可见性。

---

## 活动管理

路径：`/admin/events`

### 适合什么时候使用

- 发布活动、更新报名信息、调整可见状态
- 导出报名名单交给线下组织者或活动负责人
- 遇到已有报名记录的活动，优先修改而不是删除

### 发布活动

1. 点击"新建活动"。
2. 填写标题、摘要、正文、地点、活动时间、结束时间（可选）。
3. 可选设置最大参与人数。
4. 可选上传封面图片。
5. 点击"保存草稿"或"发布"。

### 删除活动

1. 在列表页点击删除按钮。
2. 二次确认弹窗。
3. **如果活动已有报名记录，系统会拒绝删除并返回提示。** 建议不要删除有真实报名数据的活动。

### 查看报名名单

1. 在活动列表点击"报名名单"。
2. 查看参与者姓名、联系方式、留言。

### 导出 CSV

1. 点击导出按钮下载 CSV 文件。
2. 编码为 UTF-8 BOM，防 Excel 中文乱码。
3. 以 `=`、`+`、`-`、`@` 开头的字段前加单引号，防公式注入。

### 发布/下架

列表页切换状态开关，控制前台可见性。

## 校友名单管理

路径：`/admin/alumni`

### 适合什么时候使用

- 新增或修正校友数据
- 批量导入更新名单
- 导出名单做线下校对或备份

### 查看名单

- 列表显示所有已登记的校友。
- 支持按姓名、班级搜索。

### 新增校友

1. 点击"新增"。
2. 填写姓名、届别（如"2022级3班"）、标签。
3. 标签格式：`大学名 | 专业 | 城市`（用 `|` 分隔，顺序固定）。

### 编辑校友

1. 点击行内编辑按钮。
2. 修改信息后保存。
3. 编辑后自动同步到公开搜索 API 和地图 API。

### 删除校友

1. 点击行内删除按钮。
2. 二次确认后删除，不可撤销。

### CSV 导入

1. 点击导入按钮，选择 CSV 文件。
2. 编码要求：UTF-8。
3. 字段头支持中文（姓名、届别、标签）或英文（name、graduationClass、tags）。
4. 导入去重规则：按 `name + graduationClass` 合并。

### CSV 导出

1. 点击导出按钮下载。
2. 文件名格式：`alumni-roster-YYYYMMDD.csv`。
3. UTF-8 BOM 编码，防公式注入处理。

## 校友信息修改申请审核

路径：`/admin/alumni-corrections`

### 适合什么时候使用

- 处理校友提交的姓名、届别、标签修正申请
- 审核前先比对“当前值”和“申请值”是否合理
- 如果有疑问，先驳回并留言说明原因

### 流程说明

- 用户在前台 `/alumni/correction` 搜索自己的姓名，提交修改申请。
- 申请不会直接修改数据库，需管理员审核。

### 审核步骤

1. 在列表中展开申请。
2. 查看当前值（申请前的数据）与申请值（用户希望修改为的数据）的对比。
3. 选择操作：
   - **通过并应用**：自动更新 WhitelistRoster 中对应校友的信息。
   - **驳回申请**：拒绝修改，数据不受影响。
4. 审核后不可撤销。

### 筛选与搜索

- 按状态筛选：全部 / 待审核 / 已通过 / 已驳回。
- 按姓名或联系方式搜索。

### 管理员备注

审核时可选填写备注，供记录审核原因。

## 日常操作建议

### 每天登录后先做什么

1. 查看后台总览是否有异常统计。
2. 检查待审核的修改申请和未处理投稿。
3. 查看活动报名是否有新增数据。
4. 如有新闻或活动更新，先保存草稿，再发布。

### 进行高风险操作前

- 确认当前有可用备份
- 确认是否需要先通知其他管理员
- 确认该操作是否会影响前台访问或报名数据
- 先小范围测试，再执行正式操作

### 什么时候不要直接删除

- 活动已有报名记录时
- 校友名单已被多处引用时
- 不确定数据是否已经同步到前台时

### 出现问题先看哪里

- 登录异常：先看 `/admin/login`、cookie 和环境变量
- 数据不一致：先看数据库和最近备份
- 上传失败：先看上传目录权限和磁盘空间
- 页面空白：先看浏览器控制台和 `docs/TROUBLESHOOTING.md`

## 误操作处理

### 如果不小心删除了重要数据

1. **立即停止操作**，不要继续修改其他内容。
2. **备份当前数据库**，防止二次损坏。
3. 从最近的备份中恢复数据（参见 [数据备份指南](BACKUP_GUIDE.md)）。
4. 确认数据完整性后再恢复服务。

### 原则

- 不要在没有备份的情况下执行高风险操作（批量删除、导入覆盖等）。
- 有真实用户报名数据的活动不建议删除。
- 删除操作不可撤销，操作前务必确认。
